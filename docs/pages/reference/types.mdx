# Types & Constants

Everything the package exports besides the [`Rollup`](/reference/rollup) class. All types are importable with `import type`:

```ts twoslash
import type {
    AddressLike,
    AdvanceRequest,
    BytesLike,
    GioResponse,
    InspectRequest,
    RollupRequest,
    RunHandlers,
    U256Like,
} from '@tuler/node-libcmt';
```

## Input types

Three "like" types describe what the binding accepts and converts automatically — see [byte and number conventions](/guide/emitting-outputs#byte-and-number-conventions):

| Type | Accepts | Used for |
| --- | --- | --- |
| `BytesLike` | 0x-prefixed hex `string`, `Buffer`, `Uint8Array` | payloads, gio ids |
| `AddressLike` | 0x-prefixed hex `string`, `Buffer`, `Uint8Array` (20 bytes) | voucher destinations |
| `U256Like` | `bigint`, `number`, hex `string`, `Uint8Array` (32 bytes) | voucher values |

## Request types

[`finish`](/reference/finish) returns `RollupRequest`, the discriminated union of:

- `AdvanceRequest` — `type: 'advance'` plus the input metadata (`chainId`, `appContract`, `msgSender`, `blockNumber`, `blockTimestamp`, `prevRandao`, `index`) and `payload`.
- `InspectRequest` — `type: 'inspect'` and `payload`.

```ts twoslash
import { Rollup } from '@tuler/node-libcmt';
const rollup = new Rollup();
// ---cut---
const request = rollup.finish();
if (request.type === 'advance') {
    request.index;
    //       ^?
}
```

## Handler types

`RunHandlers` is the parameter of [`run`](/reference/run) — an optional `advance` and `inspect` handler, each receiving `(request, rollup)` and returning `boolean | void`, possibly wrapped in a `Promise`.

## Response types

`GioResponse` is returned by [`gio`](/reference/gio): `{ responseCode: number, responseData: Buffer }`.

## Errors

Failed libcmt calls throw an `Error` with two extra properties:

| Property | Type | Meaning |
| --- | --- | --- |
| `errno` | `number` | The negative errno from libcmt (e.g. `-16` for `EBUSY`) |
| `syscall` | `string` | The libcmt call that failed (e.g. `cmt_rollup_init`) |

Argument validation failures throw plain `TypeError` / `RangeError` before reaching the device.
