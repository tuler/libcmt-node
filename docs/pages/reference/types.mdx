# Types & Constants

Everything the package exports besides the [`Rollup`](/reference/rollup) class and the standalone codec functions (the `RollupError` class is documented under [Errors](#errors)). All types are importable with `import type`:

```ts twoslash
import type {
    Advance,
    AddressLike,
    AdvanceRequest,
    BytesLike,
    DelegateCallVoucher,
    Hex,
    InspectRequest,
    RollupRequest,
    RunHandlers,
    U256Like,
    Voucher,
} from '@tuler/node-libcmt';
```

## Input types

Three "like" types describe what the binding accepts and converts automatically — see [byte and number conventions](/guide/emitting-outputs#byte-and-number-conventions). Hex strings are `Hex` (`` `0x${string}` ``), structurally identical to viem's `Hex`/`Address`:

| Type | Accepts | Used for |
| --- | --- | --- |
| `BytesLike` | `Hex`, `Buffer`, `Uint8Array` | output and report payloads |
| `AddressLike` | `Hex`, `Buffer`, `Uint8Array` (20 bytes) | voucher destinations |
| `U256Like` | `bigint`, `number`, `Hex`, `Uint8Array` (32 bytes) | voucher values |

## Encoder argument types

The two voucher encoders take a named options object, importable for typing helpers that build outputs:

- `Voucher` — argument of [`encodeVoucher`](/reference/encode-voucher): `destination` (`AddressLike`), optional `value` (`U256Like`, default `0n`) and `payload` (`BytesLike`, default empty).
- `DelegateCallVoucher` — argument of [`encodeDelegateCallVoucher`](/reference/encode-delegate-call-voucher): `destination` (`AddressLike`) and optional `payload` (`BytesLike`). No `value` — `DELEGATECALL` cannot transfer ether.

```ts twoslash
import type { Voucher, DelegateCallVoucher } from '@tuler/node-libcmt';

const voucher: Voucher = {
    destination: '0xf39Fd6e51aad88F6F4ce6aB8827279cffFb92266',
    value: 1_000_000_000_000_000_000n,
};

const delegateCall: DelegateCallVoucher = {
    destination: '0x5FbDB2315678afecb367f032d93F642f64180aa3',
};
```

## Request types

[`waitForInput`](/reference/wait-for-input) returns `RollupRequest`, the discriminated union of:

- `AdvanceRequest` — `type: 'advance'` and the **raw, undecoded** `payload` (`Buffer`). Parse it with [`decodeAdvance`](/reference/decode-advance) to get the input fields.
- `InspectRequest` — `type: 'inspect'` and the raw query `payload` (`Buffer`).

```ts twoslash
import { Rollup } from '@tuler/node-libcmt';
const rollup = new Rollup();
// ---cut---
const request = rollup.waitForInput();
if (request.type === 'advance') {
    request.payload;
    //      ^?
}
```

## Decoded advance type

`Advance` is the return type of [`decodeAdvance`](/reference/decode-advance) — the structured fields of an `EvmAdvance` input:

```ts twoslash
import type { Advance } from '@tuler/node-libcmt';
// ---cut---
declare const advance: Advance;
advance.msgSender;
//      ^?
```

It carries `chainId`, `appContract`, `msgSender`, `blockNumber`, `blockTimestamp`, `prevRandao`, `index` and `payload`.

## Handler types

`RunHandlers` is the parameter of [`run`](/reference/run) — an optional `advance` and `inspect` handler, each receiving `(request, rollup)` and returning `boolean | void`, possibly wrapped in a `Promise`.

## Errors

Failed libcmt calls throw a `RollupError` (a subclass of `Error`) with two extra properties:

| Property | Type | Meaning |
| --- | --- | --- |
| `errno` | `number` | The negative errno from libcmt (e.g. `-16` for `EBUSY`) |
| `syscall` | `string` | The libcmt call that failed (e.g. `cmt_rollup_init`) |

`RollupError` is exported as a value, so it can be caught with `instanceof`:

```ts twoslash
import { Rollup, RollupError } from '@tuler/node-libcmt';

try {
    new Rollup();
} catch (error) {
    if (error instanceof RollupError) {
        error.errno;
        //    ^?
    }
}
```

Argument validation failures throw plain `TypeError` / `RangeError` before reaching the device.
